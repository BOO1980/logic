Step 2 - devising and carrying out the plan

  • If the current problem resemebles a problem you have seen before, write down how it is similar
  • Write down how the problem differs from any of the previous
  • For those aspects that are similar, can you use the same solution techniques that you used in the past
  • Most programming tasks will usually compromise of a number of smaller subproblems
  • some subprobkems may be eaiser to solve than others
  • Start with eaiser tasks and they may give clues to how to do the other tasks
  • In understanding a problem, a drawing or sketch or alternative description of the problem may help
  • Solve problem
    • Providing your program with the data it needs
    • processing the data
    • Displaying the results of saving them to file
  • Exploring processing the data problem will likely yeild a further set of subproblems
  • Make a drawing, sketch or an alternative description to the problem
  • retelling a problem in your own words to another person is also affective

Step 3 - carrying out the attack

  • Once you understand the solution you need to write down the sequence of actions necessary to solve the problem
  • You need to ask yourself does the order rely on certain conditions to be trye
  • If you are finding it too complicated to understand a problem , try removing some details to see if simlifiyinf gives you a clear understanding
  • Once yoou have a sequence of actions, check whether they all need to be carried out in all cicumstances, perhaps some actions only need to be carried out in all circumstances
  • perhaps some of the actions only need to be carried out when certain conditions met
  • Is it sufficient to carry out each action just once, or does some actions needs to be repeated
  • sleeping on a problem can help, or take a walk if you find your self stuck
